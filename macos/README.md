# Claude Limits

Меню-бар приложение для macOS: показывает использование лимитов подписки Claude
(5-часовое окно, недельное, промо-лимиты вроде Fable) прямо в строке меню —
`●10% 5h·●23% 7d·●39% Fable`, точка окрашена по уровню каждого лимита
(зелёный < 50%, жёлтый 50–74%, оранжевый 75–89%, красный ≥ 90%). Всё локально:
без телеметрии и сторонних зависимостей.

## Установка

```bash
./scripts/make_app.sh --install
open ~/Applications/Claude\ Limits.app
```

При первом запуске приложение попросит разрешение на уведомления. Автозапуск —
галка «Запускать при входе» в меню.

## Как это работает

- Кредо берётся из macOS Keychain (запись Claude Code), usage — из
  `api.anthropic.com/api/oauth/usage`; опрос раз в 60 секунд и при пробуждении Мака.
- Уведомления о сбросе окна планируются заранее на момент сброса, поэтому
  срабатывают вовремя даже если Мак был офлайн в этот момент.
- При 100% лимита приходит немедленное уведомление «лимит исчерпан» (один раз
  на окно, дедуп переживает перезапуск приложения).
- Галка «Уведомления о лимитах» выключает и то и другое.

## Если в меню «Токен истёк»

Просто откройте Claude Code — он сам обновит токен в Keychain, приложение
подхватит его на следующем опросе. Ничего настраивать не нужно.

## Удаление

Сначала снимите галку «Запускать при входе» в меню приложения (иначе в
Системных настройках → «Объекты входа» останется осиротевшая запись), затем:

```bash
rm -rf ~/Applications/Claude\ Limits.app
defaults delete com.vladlaiho.claude-limits
```

## Разработка

```bash
swift build                              # сборка
swift run checks                         # ассерты по fixtures/usage_sample.json (из корня репо)
swift build -c release && .build/release/claude-limits --check   # e2e: keychain + живой API
```

Вне `.app`-бандла уведомления и «Запускать при входе» отключены
(предупреждение в stderr) — это нормальный dev-режим.
