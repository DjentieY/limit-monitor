# Limit Monitor

Меню-бар приложение для macOS: показывает использование лимитов подписок
Claude, Codex и Cursor (5-часовое окно, недельное, промо-лимиты вроде Fable,
биллинг-бакеты Cursor), а также балансы и квоты кастомных провайдеров из
`providers.json` (OpenRouter, DeepSeek, GLM и др.) прямо в строке меню —
`5h●10% │ 7d●23% │ 7d●39% Fable`,
а при нескольких активных провайдерах — группами:
`Cl·5h●45% │ 7d●30% ‖ Cx·5h●12% │ 7d●40% ‖ Cu·Auto●2% │ API●6% ‖ OR·●$74.75`.
Точка окрашена по уровню каждого лимита (зелёный < 50%, жёлтый 50–74%,
оранжевый 75–89%, красный ≥ 90%). Всё локально: без телеметрии и сторонних
зависимостей.

## Установка

```bash
./scripts/make_app.sh --install
open ~/Applications/Limit\ Monitor.app
```

`--install` убирает и старое `Claude Limits.app` (v0.1), и прежнюю копию
`Limit Monitor.app`. При первом запуске приложение попросит разрешение на
уведомления. Автозапуск — галка «Запускать при входе» в меню. При апгрейде
с v0.1 автозапуск не переносится (регистрация привязана к bundle id):
включите галку заново и удалите осиротевшую запись «Claude Limits» в
Системных настройках → «Объекты входа».

## Как это работает

- Claude: кредо из macOS Keychain (запись Claude Code), usage — из
  `api.anthropic.com/api/oauth/usage`; опрос раз в 60 секунд.
- Codex: токен из `~/.codex/auth.json` (или `$CODEX_HOME/auth.json`), usage —
  из `chatgpt.com/backend-api/wham/usage` (fallback `codex/usage`); опрос раз
  в 180 секунд, заголовки как у браузера. `auth.json` в API-key режиме —
  строка «Codex: API-key режим — план-лимитов нет».
- Cursor: токен из локальной `state.vscdb` (строго read-only через
  `/usr/bin/sqlite3 -readonly`, база никогда не копируется), usage — из
  `cursor.com/api/usage-summary`; опрос раз в 300 секунд. Окон нет — бакеты
  Auto+Composer / API-модели / on-demand сбрасываются в конце биллинг-цикла;
  безлимитный план — один зелёный `∞`.
- Кастомные провайдеры из `~/.config/limit-monitor/providers.json` — балансы
  и квоты OpenRouter, DeepSeek, Kimi, GLM Coding Plan, SiliconFlow, Novita и
  почти любого другого API через `generic-http`; сегменты вида `OR·●$74.75`,
  опрос раз в 300 секунд по умолчанию. Формат файла, ключи через Keychain и
  матрица поддержки — в корневом README (раздел «Custom providers»).
- Провайдеры независимы: ошибка или устаревшие данные одного не трогают
  других; при пробуждении Мака опрашиваются все.
- Уведомления о сбросе окна планируются заранее на момент сброса, поэтому
  срабатывают вовремя даже если Мак был офлайн в этот момент.
- При 100% лимита приходит немедленное уведомление «лимит исчерпан» (один раз
  на окно, дедуп переживает перезапуск приложения).
- Галка «Уведомления о лимитах» выключает и то и другое.
- Токены никогда не обновляются и не печатаются — только чтение.

## Настройки и виджет

- Пункт меню «Настройки…» (⌘,) открывает окно с чекбоксами провайдеров:
  встроенные Claude / Codex / Cursor и каждая запись providers.json (записи с
  `enabled: false` показаны серыми — их включают в самом файле). Снятая галка
  действует сразу: опрос останавливается, провайдер исчезает из бара, меню,
  уведомлений и снапшота; поставленная — немедленный опрос. Если выключить
  всех, в баре остаётся статичное `LM`, меню и настройки доступны.
- Раздел «Общие» дублирует переключатели меню («Уведомления о лимитах»,
  «Запускать при входе», «Виджет на рабочем столе») — синхронно в обе стороны.
- «Виджет на рабочем столе» — карточка поверх обоев (чуть выше иконок рабочего
  стола, ниже обычных окон): провайдеры, цветные точки, значения и времена
  сброса. Перетаскивается за фон, позиция запоминается. Настоящий
  WidgetKit-виджет требует Apple-подписанной сборки — рецепт законсервирован
  в `research/widget.md`.
- После каждого опроса приложение атомарно пишет снапшот
  `~/Library/Application Support/limit-monitor/widget-snapshot.json`
  (числа, метки и ISO-времена — никаких секретов). `limit-monitor --status` —
  человекочитаемая таблица из него, `--status --json` — файл как есть; оба без
  сети (интеграция для агентов/SwiftBar/Raycast). Нет снапшота → exit 2 и
  подсказка запустить приложение или `--check`.

## Если в меню «Токен истёк»

Claude: просто откройте Claude Code — он сам обновит токен в Keychain.
Codex: запустите любую команду `codex`. Cursor: откройте Cursor. Приложение
подхватит свежий токен на следующем опросе. Ничего настраивать не нужно.

## Удаление

Сначала снимите галку «Запускать при входе» в меню приложения (иначе в
Системных настройках → «Объекты входа» останется осиротевшая запись), затем:

```bash
rm -rf ~/Applications/Limit\ Monitor.app ~/Applications/Claude\ Limits.app
defaults delete com.vladlaiho.limit-monitor
defaults delete com.vladlaiho.claude-limits
```

## Разработка

```bash
swift build                              # сборка
swift run checks                         # ассерты по fixtures/*.json (из macos/)
swift build -c release && .build/release/limit-monitor --check   # e2e: keychain + живые API
.build/release/limit-monitor --status          # таблица из снапшота, без сети
.build/release/limit-monitor --status --json   # снапшот как есть (для скриптов)
```

Вне `.app`-бандла уведомления и «Запускать при входе» отключены
(предупреждение в stderr) — это нормальный dev-режим.
